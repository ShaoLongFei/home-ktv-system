<objective>
Close the Phase 5 runtime wiring gaps so the real API server can discover online candidates, persist supplement requests, and advance selected/retried tasks through the cache-before-play worker path.

Purpose: The implemented Phase 5 pieces are mostly present but disconnected in `createServer()`. This plan connects them behind an explicit safe provider/config boundary while preserving local-first behavior, no direct online playback, and no auto-enqueueing.
Output: One deterministic runtime online provider boundary, real server route injection for search and supplement requests, selected/retry worker triggering, and createServer-level regression tests.
</objective>
