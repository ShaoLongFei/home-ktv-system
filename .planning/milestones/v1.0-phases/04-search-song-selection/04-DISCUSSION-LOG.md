## 搜索召回与排序

### 搜索默认范围

| Option | Description | Selected |
|--------|-------------|----------|
| 全字段搜索 | 歌名、歌手、拼音、首字母、别名、`searchHints` 都参与，最符合 KTV 点歌习惯。 | yes |
| 先搜歌名/歌手 | 用户切换高级搜索才搜拼音和别名。 | |
| 只做歌名/歌手 | 拼音和别名后续再补。 | |

**User's choice:** 全字段搜索

### 排序优先级

| Option | Description | Selected |
|--------|-------------|----------|
| 明确命中优先 | 歌名精确 > 歌手精确 > 标准化/模糊歌名 > 别名 > 拼音 > 首字母 > `searchHints`。 | yes |
| 热门/权重优先 | 命中后按 `searchWeight` 明显提前热门歌。 | |
| 最近/本地优先 | 更偏家庭常唱和本地可播稳定性。 | |

**User's choice:** 明确命中优先

### 首字母搜索排序

| Option | Description | Selected |
|--------|-------------|----------|
| 召回但降权 | 能搜到，但不压过中文精确歌名和歌手命中。 | yes |
| 首字母强优先 | `qlx` 这类输入时首字母命中排最前。 | |
| 输入长度门槛 | 只有输入长度达到 2-3 位才启用首字母召回。 | |

**User's choice:** 召回但降权

### 搜索容错范围

| Option | Description | Selected |
|--------|-------------|----------|
| 轻量容错 | 大小写、空格、符号、简繁、全半角、拼音连续串处理好。 | yes |
| 强模糊 | 允许错字、近似音、编辑距离匹配。 | |
| 保守精确 | 只做标准化和包含匹配，避免误召回。 | |

**User's choice:** 轻量容错

### 默认推荐版本

| Option | Description | Selected |
|--------|-------------|----------|
| 沿用正式歌库 defaultAsset 所在版本 | 与后台维护结果一致。 | |
| 优先最高清/最新资源 | 默认推荐更偏向用户感知质量。 | yes |
| 优先最近成功播放过的版本 | 依赖历史数据。 | |
