# Phase 06 UAT

## 目标

确认 TV 端在真实浏览器和电视距离下，能清楚表达空闲、播放、首播拦截、切歌、原唱/伴唱切换、恢复、离线和冲突状态。

## 启动环境

1. 启动数据库和服务：
   ```bash
   DATABASE_URL=postgres://ktv:ktv@127.0.0.1:5432/home_ktv \
   MEDIA_ROOT=/Users/shaolongfei/OtherProjects/home-ktv-system/home-ktv-media \
   PUBLIC_BASE_URL=http://10.91.130.150:4000 \
   pnpm dev:local restart
   ```
2. 记录命令输出里的 TV 地址，优先打开 `TV LAN` 或 `TV local`。
3. 如果浏览器首次播放被拦截，先不要处理错误，继续按下面步骤验证首播提示。

## 验证步骤

1. 打开 TV 页面，确认空闲态显示 `扫码点歌`，并且大二维码下方有 `手机扫码点歌`。
2. 检查 idle 页面、二维码和文案没有互相遮挡。
3. 在手机端点第一首歌后，如果 TV 显示 `点击电视开始播放`，点击 TV 页面一次。
4. 确认首播提示消失，歌曲开始播放。
5. 确认播放态同时显示当前歌曲、歌手、原唱/伴唱模式、状态标签、`mm:ss / mm:ss`、下一首和小二维码。
6. 在手机端切换原唱/伴唱，确认 TV 上模式标签同步变化；如果发生回退，确认显示 `切换失败，已恢复到原模式`。
7. 在手机端切歌，确认 TV 只给出短反馈，不会盖住当前歌曲、时间和下一首。
8. 刷新 TV、停止 API、或让房间进入冲突状态，确认 TV 能显示离线/恢复/冲突文案，且主状态清楚可读。
9. 运行视觉检查：
   ```bash
   pnpm tv:visual-check
   ```
10. 确认生成以下文件：
    - `logs/visual/tv-player-1920x1080.png`
    - `logs/visual/tv-player-1366x768.png`
11. 打开两张截图，确认没有二维码、状态、时间和歌名重叠，且长标题没有把时间挤出底栏。

## 通过标准

- 空闲、播放中、首播拦截、切歌、切换、恢复、离线和冲突状态都能看懂。
- `点击电视开始播放` 会在首次播放受阻时出现，并在点击 TV 后消失。
- `mm:ss / mm:ss` 始终可见。
- `pnpm tv:visual-check` 产出两张有效截图。
