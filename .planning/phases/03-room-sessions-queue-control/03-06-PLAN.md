Create `use-room-controller.ts`:
- Parse `room` and `token` from URL.
- Always try `GET /rooms/:roomSlug/control-session?deviceId=...` first.
- If restore fails and token exists, call `POST /rooms/:roomSlug/control-sessions`.
- If token POST returns `INVALID_PAIRING_TOKEN`, retry restore before showing an error.
- After any successful cookie restore or token exchange, call `history.replaceState` to remove `token` when the URL contains `token`.
- Open `/rooms/:roomSlug/realtime?deviceId=${deviceId}&client=mobile`.
- Set `fallbackPollingIntervalMs = 5000`.
- Set `sessionRefreshIntervalMs = 15 * 60 * 1000`.
- While WebSocket is connected, call `GET /rooms/:roomSlug/control-session?deviceId=...` every `sessionRefreshIntervalMs` to renew the httpOnly cookie Max-Age.
- On WebSocket failure, show reconnect state and poll restore every 5000ms.
- On `SESSION_VERSION_CONFLICT`, replace local snapshot with server snapshot and do not auto-replay stale commands.
- Render `撤销` only from server-authored undo metadata: `RoomQueueEntryPreview.undoExpiresAt` or delete response `undo.undoExpiresAt`.
