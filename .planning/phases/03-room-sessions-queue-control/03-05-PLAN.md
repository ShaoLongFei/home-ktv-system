  <action>
Install `@fastify/websocket@11.2.0` in `apps/api/package.json` and update `pnpm-lock.yaml`.

Create `room-snapshot-broadcaster.ts`:
- `subscribe(roomSlug, connection)`.
- `unsubscribe(roomSlug, connection)`.
- `broadcastRoomSnapshot(roomSlug, snapshot)`.
- Envelope type must be `"room.control.snapshot.updated"`, `roomId = snapshot.roomId`, `version = snapshot.sessionVersion`, `timestamp = snapshot.generatedAt`, and `payload = snapshot`.

Create `routes/realtime.ts`:
- Register `@fastify/websocket`.
- Export `MOBILE_REALTIME_RENEW_INTERVAL_MS = 60 * 1000`.
- Implement `GET /rooms/:roomSlug/realtime?deviceId=...&client=mobile|tv|admin`.
- For `client=mobile`, require `ktv_control_session` cookie plus matching `deviceId` before sending a `RoomControlSnapshot`.
- On accepted mobile connection, call `touchControlSession` so realtime activity renews idle expiry.
- On accepted mobile connection, start a renewal interval that calls `touchControlSession` every `MOBILE_REALTIME_RENEW_INTERVAL_MS`.
- Stop the renewal interval when the socket closes.
- WebSocket renewal updates server-side `lastSeenAt` and `expiresAt`; browser `ktv_control_session` cookie Max-Age renewal is handled by the mobile app's periodic HTTP restore in Plan 03-06.
- On connect, build and send current `RoomControlSnapshot`.
- Send `{ type: "ping", timestamp }` every 30000ms and clear interval on close.
- Reject unknown rooms before subscribing.

Update `server.ts` to create one `RoomSnapshotBroadcaster` per server instance and register realtime routes.
  </action>
