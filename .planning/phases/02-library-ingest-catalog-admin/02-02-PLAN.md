Implement these exact routes:
- `GET /admin/import-candidates` with optional comma-separated `statuses`; default statuses are `pending,held,review_required,conflict`. Response shape: `{ candidates: Array<{ candidate, files }> }`.
- `POST /admin/scans` with body `{ "scope": "imports" | "songs" | "all" }`; call `enqueueManualScan(scope)` and return `{ status: "queued", scope }`.
- `PATCH /admin/import-candidates/:candidateId/metadata` accepting `{ title, artistName, language, genre, tags, aliases, searchHints, releaseYear, defaultVocalMode, sameVersionConfirmed, files }`, where `files` contains `{ candidateFileId, selected, proposedVocalMode }`.
- `POST /admin/import-candidates/:candidateId/hold` accepting `{ note?: string }`; set status to `held` and keep file rows under `imports/needs-review`.
- `POST /admin/import-candidates/:candidateId/reject-delete` accepting `{ confirmDelete: true }`; if missing or false, return 400 `{ code: "CONFIRM_DELETE_REQUIRED" }`; when true, call `fs.rm(absoluteFilePath, { force: true })` for each selected file and mark status `rejected_deleted`.
- `POST /admin/import-candidates/:candidateId/approve` accepting `{ conflictResolution?: "merge-existing" | "create-version", targetSongId?: string }`; approve only through `CatalogAdmissionService.approveCandidate`.

Extend `CatalogAdmissionService` with `approveCandidate(input)`:
- Load candidate and selected files.
- Require exactly one selected `original` and one selected `instrumental`.
- Call `evaluateSwitchPair` with `sameVersionConfirmed`, durations, and selected count.
- When decision is `held`, set candidate status `held` and return 422 `{ code: "MISSING_COUNTERPART" }`.
- When decision is `review_required`, set status `review_required` and return 422 `{ code: "REVIEW_REQUIRED", reason }`.
- When decision is `rejected`, set status `review_required` and return 422 `{ code: "ADMISSION_REJECTED", reason }`; do not delete files automatically.
- For verified approval, plan target directory exactly `${MEDIA_ROOT}/songs/${language}/${artistName}/${title}/`.
- If target directory exists and `conflictResolution` is absent, set candidate status `conflict` and return 409 `{ code: "FORMAL_DIRECTORY_CONFLICT" }`; do not merge automatically per D-02.
- Move files to `original.mp4` and `instrumental.mp4`, write `song.json`, insert `songs` row, insert two `assets` rows with `source_type = 'local'`, `asset_kind = 'video'`, `status = 'ready'`, `switch_quality_status = 'verified'`, same `switch_family`, and set `songs.default_asset_id` to the instrumental asset per D-14.
- Insert `source_records` rows with `provider = 'local-import'` and `original_path` equal to the import relative path.

Add repository methods needed by the routes with concrete names: `createApprovedSong`, `createApprovedAsset`, `setDefaultAsset`, `listCandidateFiles`, `updateCandidateFileRoles`, and `markRejectedDeleted`.
