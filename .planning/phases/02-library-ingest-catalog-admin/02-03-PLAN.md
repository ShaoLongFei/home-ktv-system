Create `CatalogAdmissionService` in `apps/api/src/modules/catalog/admission-service.ts`. It must own the formal promotion workflow and expose:
- `evaluatePair(input)` returning `verified`, `review_required`, or `rejected` with a reason. A duration delta `>300ms` must never become `verified`, even with manual confirmation, per D-12.
- `holdCandidate(candidateId)` that resolves each `ImportCandidateFileDetail` under the configured media root, physically moves files from `imports/pending` into `imports/needs-review`, updates each `import_files.root_kind` to `imports_needs_review`, updates relative paths if the move changes them, and sets candidate status `held`.
- `rejectDeleteCandidate(candidateId, { confirmDelete: true })` that deletes physical files and marks rows deleted/rejected. Without confirmation, return a 400 route error.
- `approveCandidate(candidateId)` that verifies one original plus one instrumental, same-version confirmed, probeable durations, and duration delta `<=300ms`; then computes target directory `${MEDIA_ROOT}/songs/${language}/${artistName}/${title}/`, handles D-02 conflicts before any file move or `song.json` write, moves files to the chosen formal directory, writes `song.json`, inserts/updates `songs` and `assets`, creates `source_records`, sets instrumental as default asset per D-14, and marks candidate approved.
- `resolveCandidateConflict(candidateId, input)` accepting `{ resolution: "merge_existing" | "create_version", targetSongId?: string, versionSuffix?: string }` and returning the same approved/conflict/approval_failed shape as approve.

Implement D-02 conflict handling with no automatic merge:
- Before moving files or writing `song.json`, compute `${MEDIA_ROOT}/songs/${language}/${artistName}/${title}/`.
- If the target directory exists and the request lacks explicit conflict resolution, set candidate `status = 'conflict'` and write conflict metadata into `candidate_meta`, including `targetDirectory`, `conflictType`, and existing matched `songId` if known.
- Return route error code `FORMAL_DIRECTORY_CONFLICT` with the candidate id, status, and conflict metadata so the admin UI can render resolution controls.
- `merge_existing` must require `targetSongId` and still run strict pair admission before moving/writing into the existing song directory.
- `create_version` must require a non-empty `versionSuffix`, promote to a distinct directory such as `${MEDIA_ROOT}/songs/${language}/${artistName}/${title}-${versionSuffix}/`, and still run strict pair admission.
- Do not merge, overwrite, or rename automatically when the directory conflict is found.

Implement idempotent promotion and repair handling:
- Use deterministic target paths and switch family ids so reruns can identify already-moved files and already-created DB rows.
- If files moved but DB write fails, rerun must detect files in `/songs` and complete DB/song.json repair instead of failing because source files are gone.
- If DB rows exist but `song.json` is missing or invalid, rerun must rewrite `song.json`.
- On unrecoverable mismatch, set candidate status `approval_failed` and store a repair reason in `review_notes` or `candidate_meta`.

Create `song-json.ts` to write the formal `/songs` `song.json` with metadata, assets, default asset, vocal modes, durations, and switch family. Use atomic temp-file write plus rename inside the target directory.
